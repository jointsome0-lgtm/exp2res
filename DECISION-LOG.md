- 2026-07-11 — Issue #11 evidence classification: §10 owns the ten-value `EvidenceStrength` taxonomy, `EvidenceItem.strength` is typed by it, and Stage 2 uses `commit_or_pr` for the same GitHub commit as §19.3; `EvidenceItem.evidence_type` is removed because source origin already lives on `RawLog` and the field had no distinct values or consumers. Rejected alternative: introduce a second `EvidenceType` taxonomy — it would duplicate the strength/origin axes and enlarge v1 without a demonstrated use.
- 2026-07-11 — Issue #11 fact-source relations: `fact_sources.support_type` is constrained to `direct` or `corroborating`, independently of `EvidenceStrength`, and every fact requires a direct source. Rejected alternative: leave an open string or reuse evidence-strength values — either preserves an undefined value space or collapses source relationship and source strength into one axis.
- 2026-07-11 — Issue #16 temporal shape: `OccurredAt.precision` is the sole discriminator, its value determines the legal `start`/`end` shape, and the redundant `OccurredAt.kind`, `OccurredAtKind`, and `occurred_kind` storage column are removed. Rejected alternative: persist both fields plus a legal-pair matrix — it stores derivable state and adds a consistency surface with no v1 information gain.
- 2026-07-11 — Issue #16 confidence axes: `Confidence` types general confidence on `ExperienceFact`, `SelfSignal`, and `SelfClaim`, while `TemporalConfidence` is reserved for `OccurredAt.confidence`. Rejected alternative: reuse `TemporalConfidence` because the members currently match — it conflates confidence in when something happened with confidence in what a derived claim asserts.
- 2026-07-11 — Issue #15 ownership comparison: `OwnershipLevel` is a normative total order from `unknown` through `led`, with `unknown` weakest and claim confidence kept independent; verifiers fail closed when ownership language cannot be normalized. Rejected alternative: make `unknown` incomparable — a verifier could not conservatively decide whether any known ownership phrase is an upgrade.
- 2026-07-11 — Issue #15 temporal comparison: non-range `TemporalPrecision` values compare by a weak-to-strong rank and normalized maximum uncertainty width, while ranges compare by actual width with approximate bounds weaker at equal width. Rejected alternative: compare raw enum positions — range precision depends on duration, so a fixed position would misclassify cases such as a two-day range versus a month.

Rejected alternative: define a precision-implied width for zero-width ranges — it would give one moment two representations and make §16.7's equal-width exactness rule ambiguous.
